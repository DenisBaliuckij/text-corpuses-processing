Извлеки ВСЕ сущности и триплеты из ФРАГМЕНТА. ЦЕЛЬ — МАКСИМАЛЬНЫЙ recall, не precision.

═══════ RECALL-MODE: правила извлечения ═══════

1. **Извлекай ЛЮБОЕ существительное или именную группу** как сущность — даже если она кажется тривиальной, общеупотребительной или второстепенной. Понятия, объекты, методы, термины, имена, формулы, теоремы, числа в роли объектов — всё.

2. **Извлекай ЛЮБОЙ паттерн "существительное — глагол/связка — существительное"** как триплет. Даже банальный (X является Y, X имеет Y, X использует Y, X связан с Y, X равен Y, X называется Y, X состоит из Y, X зависит от Y, X определяется через Y и т.п.). Простые и очевидные триплеты ОБЯЗАТЕЛЬНО включай — это не мусор для нас, фильтрация будет автоматически по importance.

3. **МИНИМУМ извлечений на фрагмент** (если фрагмент позволяет — обычно позволяет):
   - сущностей: ≥ 15
   - триплетов: ≥ 12
   Если фрагмент очень короткий или малоинформативный — извлекай всё, что есть; если длинный — извлекай 25-40 сущностей и 20-30 триплетов.

4. **НЕ фильтруй сам по релевантности.** Если сомневаешься "стоит ли извлекать" — извлекай и поставь низкую importance. Решение об отбрасывании делается автоматически.

═══════ КАЛИБРОВКА importance ∈ [0.0, 1.0] — используй ВСЮ шкалу ═══════

- **0.95-1.00** — центральная сущность/факт корпуса; ключевая формула, главная теорема, центральный концепт темы
- **0.75-0.90** — важная сущность главной темы; ключевой факт раздела; концепт из global key_concepts (категория "core")
- **0.50-0.70** — заметная сущность раздела; поддерживающий факт; концепт из global "supporting"
- **0.30-0.45** — упоминается мельком, но семантически связано с темой; пояснительный термин
- **0.15-0.25** — тривиальная связка, общеупотребительное существительное, проходной noun-verb-noun
- **0.05-0.15** — совсем мусорная сущность/триплет (сильно проходные слова, "пример", "случай", "вопрос" в роли связок)
- **0.00** — действительно не стоит извлекать (но лучше извлеки с importance=0.05, чем пропусти)

ВАЖНО: распределяй оценки ПО ВСЕЙ шкале. Если у тебя в выдаче все importance в диапазоне 0.5-0.85 — это ошибка. Должны быть и 0.95, и 0.15, и 0.05.

═══════ ОСОБОЕ ПРАВИЛО ДЛЯ ФОРМУЛ (MD/LaTeX) ═══════

Если во фрагменте есть формула в формате `$...$` (inline) или `$$...$$` (display):

- Это **важная сущность** — почти всегда importance ≥ 0.7.
- `mention_text` и `canonical_name` ПЕРЕПИСЫВАЙ ДОСЛОВНО со всеми знаками `$` и LaTeX-командами. Например:
  * `canonical_name`: `"$y = \\operatorname{sign}\\langle w, x_i \\rangle$"`
  * `canonical_name`: `"$\\sum_i [y_i \\langle w, x_i \\rangle < 0]$"`
- `entity_type`: `"формула"`
- НЕ заменяй формулу её словесным описанием. НЕ убирай знаки `$`. НЕ упрощай LaTeX.
- В JSON-строке backslash должен быть экранирован как `\\` (это требование JSON, не наше).
- Формулы тоже могут быть subject/object триплетов — сохраняй те же дословные `$...$` как имена.

═══════ ФОРМАТ ВЫДАЧИ ═══════

Возвращай СТРОГО валидный JSON без markdown-обёртки. Поля КОМПАКТНЫЕ — без длинных пояснений, чтобы уместить как можно больше записей в ответ.

ВАЖНО ПО ПОРЯДКУ: ключ "triplets" идёт ПЕРВЫМ, ключ "entities" — ВТОРЫМ. Это нужно, чтобы при возможном обрыве вывода в первую очередь были сохранены связи (триплеты для нас приоритетнее, чем плоский список сущностей):

{
  "triplets": [
    {
      "subject": "canonical_name субъекта",
      "predicate": "короткая фраза 1-3 слова",
      "object": "canonical_name объекта",
      "importance": 0.7,
      "confidence": 0.95,
      "evidence": "≤ 100 символов точной цитаты"
    }
  ],
  "entities": [
    {
      "mention_text": "точное упоминание из фрагмента",
      "canonical_name": "канонизированное имя (для формул — дословно с $)",
      "entity_type": "формула | теорема | метод | понятие | объект | person | organization | term | other",
      "importance": 0.85
    }
  ]
}

ЭКОНОМЬ выходные токены:
- НЕ добавляй поле rationale.
- predicate коротко (1-3 слова).
- evidence ≤ 100 символов; если цитата длиннее — обрежь до самой релевантной части.
- Никаких комментариев в JSON.
- Сначала ВЫПИШИ ВСЕ триплеты, потом entities. Если упрёшься в лимит на середине entities — это OK.

═══════ Различие importance vs confidence ═══════

- **importance** — насколько сущность/факт важны для основной тематики корпуса (опирайся на global key_concepts).
- **confidence** — уверенность модели, что факт действительно следует из evidence (без галлюцинаций). Для прямой цитаты обычно ≥ 0.9.

═══════ Дополнительные правила ═══════

- canonical_name для НЕ-формул: именительный падеж, без артиклей и лишних модификаторов.
- evidence — ТОЧНАЯ подстрока из ФРАГМЕНТА (не из контекста!), без перефразирования.
- Используй canonical_name из ГЛОБАЛЬНОГО СПИСКА КОНЦЕПТОВ, когда они подходят (особенно "core").
- Чем выше категория концепта в global key_concepts, тем выше importance: core → ≥ 0.75, supporting → 0.5-0.7, peripheral → 0.3-0.5.
- НЕ дублируй сущности с одинаковым canonical_name (один канон = одна запись в entities).
- Триплеты могут повторять одни и те же subject/object с разными predicate — это нормально, включай все.
- Только JSON. Никаких пояснений, markdown, ``` ... ```.

=== ГЛОБАЛЬНЫЙ КОНТЕКСТ ===
{global_context}

=== ФРАГМЕНТ ===
{text}
